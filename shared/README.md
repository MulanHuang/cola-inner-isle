## 🗄️ database/ - 数据库数据

包含微信小程序云数据库的初始化数据和说明文档。

### 目录结构

```
database/
├── init-data/                        # 初始化数据（JSON 格式）
│   ├── meditations.json              # 冥想音频数据
│   ├── emotion-quotes.json           # 情绪文案数据
│   └── ...                           # 其他数据
├── README.md                         # 数据库设计文档
├── README_情绪文案导入说明.md         # 情绪文案导入指南
└── 冥想数据修复说明.md                # 冥想数据修复指南
```

### 主要文档

- **README.md** - 完整的数据库设计文档，包含所有集合的字段说明
- **README_情绪文案导入说明.md** - 如何导入情绪记录文案数据
- **冥想数据修复说明.md** - 冥想音频数据的修复和导入说明

### 数据集合

- `meditations.json` - 冥想音频数据（8 个脉轮冥想）
- `emotion-quotes.json` - 情绪记录温暖文案（15 条）
- 其他集合数据...

### 导入方法

#### 方法 1：通过微信开发者工具（推荐）

1. 打开微信开发者工具
2. 进入「云开发控制台」
3. 选择「数据库」
4. 找到或创建对应的集合
5. 点击「导入」按钮
6. 选择 `init-data/` 目录下的 JSON 文件
7. 确认导入

#### 方法 2：通过云开发控制台

1. 登录 [微信云开发控制台](https://console.cloud.tencent.com/tcb)
2. 选择你的环境
3. 进入「数据库」→「集合管理」
4. 找到对应的集合
5. 点击「导入」
6. 上传 JSON 文件
7. 确认导入

### 注意事项

- 导入前请先创建对应的数据库集合
- 确保 JSON 文件格式正确
- 导入后验证数据完整性
- 设置正确的数据库权限

## 🖼️ images/ - 图片资源

小程序使用的所有图片资源。

### 目录结构

```
images/
├── icons/                # 图标
├── tab/                  # TabBar 图标
├── shortcuts/            # 快捷方式图标
├── logo.svg              # Logo
└── README.md             # 图片资源说明
```

### 图片规范

详见 `images/README.md`，包含：
- 图片尺寸规范
- 格式要求
- 命名规范
- 资源获取建议

### 使用方式

#### 在小程序中使用

```xml
<!-- 本地图片 -->
<image src="/images/icons/icon.png" />

<!-- 推荐：使用云存储 -->
<image src="{{cloudImageUrl}}" />
```

**建议**：将图片上传到云存储，使用云存储 URL，可以减小小程序包体积。

#### 在 Web 应用中使用

如果 Web 应用需要使用这些图片，可以：
1. 复制到 `web/public/` 目录
2. 或上传到 CDN
3. 或使用云存储

## 🔄 资源管理建议

### 图片资源

1. **小程序**
   - 常用小图标：放在小程序目录内
   - 大图、封面：上传到云存储
   - TabBar 图标：必须放在小程序目录内

2. **Web 应用**
   - 复制需要的图片到 `web/public/`
   - 或使用 CDN

### 数据资源

1. **初始数据**
   - 保存在 `database/init-data/` 目录
   - 使用 JSON 格式
   - 添加详细的说明文档

2. **数据更新**
   - 更新 JSON 文件
   - 重新导入到云数据库
   - 或使用云函数批量更新

## 📝 添加新资源

### 添加新的数据集合

1. 在 `database/init-data/` 创建 JSON 文件
2. 在 `database/README.md` 中添加集合说明
3. 创建导入说明文档（如有必要）

### 添加新的图片资源

1. 将图片放入 `images/` 对应的子目录
2. 遵循命名规范
3. 更新 `images/README.md`（如有必要）

### 添加新的资源类型

1. 在 `shared/` 下创建新的子目录
2. 添加 README 说明
3. 更新本文档

---

## 🚨 注意事项

1. **不要提交敏感信息** - 如 API Key、密码等
2. **图片优化** - 压缩图片以减小体积
3. **版权问题** - 确保有权使用所有资源
4. **文档同步** - 添加资源时同步更新文档
5. **备份** - 重要数据定期备份
